## Collaboration Guidelines
1. **Pull Request (PR) Workflow**: All code changes must be submitted via PRs to be merged into the `main` branch.
2. **Issue Tracking**: All tasks must be managed via GitHub Issues, and every issue must be assigned to a specific member.
3. **Communication**: Technical discussions should primarily occur in GitHub Issue comments to ensure all key decisions are documented transparently.
4. **Documentation Principles**: Report sections must be submitted by group leads to the `docs/` folder according to the project timeline.
